Latent Dirichlet Allocation

Latent Dirichlet Allocation is a probabilistic topic model algorithm for discrete data, which could be applied to many scenarios like textual corpora. The assumed generative process of this model includes three steps (Blei et al., 2003):

  1. Choose \( N \sim \text{Poisson}(\xi) \)
  2. Choose \( \boldsymbol{\theta} \sim \text{Dir}(\boldsymbol{\alpha}) \)
  3. For each of the \( N \) words \( w_n \):
    • Choose a topic \( z_n \sim \text{Multinomial}(\boldsymbol{\theta}) \)
    • Choose a word \( w_n \) from \( p(w_n \mid z_n, \boldsymbol{\beta}) \), a multinomial probability conditioned on the topic \( z_n \)

UMASS Coherence

The UMASS coherence score is defined as:

$$C_{\text{UMass}}(T) = \sum_{m=2}^{M} \sum_{l=1}^{m-1} \log \frac{p(w_m, w_l) + \frac{1}{D}}{p(w_l)}$$

If the UMASS score is closer to 0, it indicates better topic coherence. The score can fluctuate around 0 depending on the number of topics \(K\) and the dataset characteristics.

The best practice is to plot UMASS against different values of \(K\) and select the number of topics where UMASS is closest to 0 (Mimno et al., 2011).

For different datasets, the UMASS coherence values give us a different number of topics because the words and the number of data sets varies.

By using UMASS coherence, we get 7. When the corpus is split to 7 individual topics, UMASS coherence score is closest to 0, meaning that this number of topics yields the best coherence.

LDA Visualization

To present the topic model of the YSL trial in an interactive format, we use pyLDAvis to generate and represent the model.

Topic Difference

The circles in the left pane of the visualization show the distance between topics, which reflects their inter-topic differences. This is calculated using the Jensen-Shannon divergence between topic-word distributions.

Overall Prevalence

The area of each circle represents the proportion of the overall prevalence of that topic in the model.

Term Relevance

pyLDAvis defines the relevance of term \( w \) to topic \( k \) given a weight parameter \( \lambda \) (where \( 0 \leq \lambda \leq 1 \)) as:

$$r(w, k \mid \lambda) = \lambda \log(\phi_{kw}) + (1 - \lambda) \log\left( \frac{\phi_{kw}}{p_w} \right)$$

If \( \lambda = 1 \), terms are ranked solely by topic-specific probability. If \( \lambda = 0 \), terms are ranked solely by lift — the ratio of a term’s probability within a topic to its marginal probability across the corpus.
